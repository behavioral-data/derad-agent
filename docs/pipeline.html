Fact-check pipeline

How one X mention becomes one tone-rendered, evidence-cited reply. Each stage shows the model it uses, the goal, a one-line prompt summary, and the tools it touches. The pipeline is now action-typed: Stage 2+3 selects one of five actions that drives Stage 4 search strategy, Stage 4.5 reconcile mode, and Stage 7 render template.

Invariance contract. Stages 1–6 produce a deep-immutable FrozenVerdict that now carries action, action_source, pivoted_from, action_outcome, and invoker_instruction_text in addition to the legacy verdict_label. The renderer at Stage 7 reads only presentation_payload + tone_neutral_justification — reasoning fields are off-limits by contract. Reply bodies contain zero URLs; all sources live on the /info page reached via the t.co short link the poster appends automatically.

action vocabulary

Five actions, one chosen per mention

Stage 2+3 selects the action. It drives the seed queries in Stage 4, the reconcile mode in Stage 4.5, and the render template in Stage 7.

verify
Falsifiable factual claim. Check against primary sources, fact-checkers, reputable news. Default action.
provide_context
Claim is literally true but the framing omits something material (cherry-picked stat, decontextualised image). Surface the missing context.
challenge_opinion
Strongly-stated opinion with falsifiable premises. Named credible critics have published push-back. Surface the counterpoints.
surface_perspectives
Genuinely contested topic; no single correct answer. Present ≥2 distinct credible camps with equal weight.
decline
Nothing actionable — pure personal-aesthetic reaction, joke, or non-sequitur. Short-circuits Stages 4–5. Bot acknowledges politely.

4
Verify (Papelo)
Action-conditional iterative search loop
Claude + web_search_20250305 (default) gpt-5.4-mini (fallback) Claude controller action-conditional caps 60 s wall-clock
goal
Gather web evidence for the chosen action. Seed queries and stop conditions are action-specific. The default search backend is Claude + web_search_20250305 on Foundry — it doesn't refuse on sensitive queries the way gpt-5.4-mini did.
seed queries
verify — 1 seed: "fact check: {claim}"
provide_context — 2 seeds: "{claim} background context", "{claim} full story"
challenge_opinion — 2 seeds: "critique of: {claim}", "evidence against: {claim}"
surface_perspectives — 2 seeds: "{claim} debate", "{claim} different perspectives"
controller
"Decide if enough evidence has been gathered for the action's stop condition. If not, issue the SINGLE next question that closes the biggest gap." Stop conditions are action-specific: verify = ≥2 fact-checker / reputable-news agree; provide_context = ≥2 sources document the missing piece; challenge_opinion = ≥2 named credible critics; surface_perspectives = ≥2 distinct camps each with ≥1 credible source.
caps
verify: ≤5 questions. All other actions: ≤6 questions (2 seeds + 4 controller steps). 3 hits per question.
search prompt
"Surface primary-source coverage; cite each result. Stay grounded in what published sources actually say — do not editorialize. Surfacing accurate sources is the task." (Claude + native web_search_20250305 tool. URLs come from web_search_tool_result blocks — fabrication impossible.)
validate + enrich
8-worker thread pool. Each hit is fetched once via _fetch_clean_page: annotation URLs (server-vouched) skip title-match check but still get body extracted; markdown URLs get HEAD + Jaccard title-match ≥ 0.20. Clean article body extracted by trafilatura (up to 3000 chars) → stored as body_markdown per hit for Stage 4.5.
output
list of Evidence(question, source_url, snippet, stance="neutral", body_markdown) — stances re-stamped by Stage 4.5

Source tier
Classify every URL into a reliability tier
structural L1 dict · L2 Tables · L3 Claude
goal
Tag every URL as fact-checker / reputable-news / primary-source / aggregator / low-quality / satirical / unknown so structural verdict and action-outcome rules can read off it.
L1
In-memory: 243 curated domains across the tiers, plus subdomain → parent fallback. ~0 ms.
L2
Azure Tables SourceTierCache — fleet-wide; survives worker/pod restarts. One classification ever per domain.
L3 prompt
"Classify each input domain into ONE tier from this exact set. Be conservative — return 'unknown' if unsure." (Claude · low reasoning · 30 s timeout; only fires on L1+L2 miss; result writes back to both caches.)
output

↓
4.5
Reconcile
Action-driven, four-mode evidence reconciliation
Claude Sonnet 4.6 reasoning_effort=medium · 4096 thinking 90 s timeout skipped on structural NEI
goal
Reason across text evidence + image OCR + image provenance + author context. Mode is set by the pipeline's action — each mode populates a different set of output buckets.
evidence
Each Evidence row carries snippet (~150 chars, the cited quote) AND body_markdown (full article body from trafilatura, up to 3000 chars). Claude uses body_markdown as the primary basis — what the publisher actually reported — and falls back to snippet only when body extraction failed (paywall, JS-only, 404).
mode: verify
Populate verified / refuted / disputed / unaddressed buckets. counter_fact when refuted. primary_sources_to_cite (1–3, fact-checker > reputable-news).
mode: provide_context
Populate contextual_findings — one entry with is_central=true. context_note (≤220 chars): the missing context the framing hides. Don't populate verified/refuted buckets.
mode: challenge_opinion
Populate challenged_propositions — 1–3 Counterpoint objects each with named credible critic, summary (≤160 chars), citing_sources, weight ∈ {strong, moderate, weak}.
mode: surface_perspectives
Populate perspectives — 2–3 entries each with label (≤22 chars, tweet-compact: "Pro-UBI", "Cost-control view"), summary (≤140 chars), citing_sources. Central proposition goes in unaddressed_propositions — topic itself can't be "verified".
shared prompt
"Use tweet_context actively — parody handles, recycled-event tweets, third-party aggregators. For image-bearing claims fold Lens 2 (image-text) + Lens 3 (cross-modal) reasoning inline into the narrative. Don't invent sources; cite only from source_quality_table. Conservative when evidence is thin."
skip when
No text evidence AND no image-provenance URL in a credible tier → structurally NEI before reconcile fires; saves ~5–10 s + the Claude call.
output
consolidated_findings · presentation_payload · tone_neutral_justification · evidence_stances (re-stamped per evidence). Stance-count drift emits reconcile_stance_drift metric.
↓

pure Python < 50 ms
goal

7
Render
5 action templates × 3 tone registers — compositional renderer
Claude Sonnet 4.6 no thinking · 30 s timeout ≤256 X-weighted chars zero URLs in body
architecture
system = _ACTION_TEMPLATES[action] + _TONE_REGISTERS[tone] + _hard_constraints_for(action, state, pivoted). Five action templates × three tone registers compose into 15 distinct prompts — not a hand-written Cartesian product.
action templates
verify: state the headline_finding, name sources by display_name, incorporate counter_fact if refuted.
provide_context: surface the missing context from context_note. Don't argue the claim is wrong.
challenge_opinion: NAME the credible critic / outlet / study. Push back on the opinion not the person.
surface_perspectives: present EXACTLY two perspectives in tension. Strict neutrality — no editorial stance. Sarcasm OFF.
decline: brief acknowledgment (≤120 chars) that there's nothing to fact-check.
tone registers
neutral = Bode & Vraga 2020 "straightforward, evidence-first, detached"; agreeable = Lewandowsky Debunking Handbook "affirm impulse, lead with alternative fact, never condescending"; satirical = Boukes & Hameleers 2022 "pointed, sardonic — target the claim or source's credibility, never the person."
URL rule
Zero URLs in reply body — hard constraint. The poster appends a t.co-shortened /info link automatically (+23 chars). The renderer names sources by display_name only (e.g. "Snopes", "AP News"). Old conditional-URL logic removed.
state-aware
State derives from action_outcome: actionable (verify supported/refuted/conflicting, context_provided, challenged, perspectives_surfaced) · no_evidence (any _nei / _unavailable / _insufficient outcome) · declined.
pivot disclosure
When pivoted_from is set: renderer is given the original ask + invoker's mention text and is instructed to weave a brief natural disclosure into the reply (e.g. "this is actually verifiable, so: …"). Model owns the full 256-char budget including any pivot clause.
enforce
Runtime invariance check: rejects renders that include any URL, exceed 256 X-weighted chars, or look like a refusal. Retries with constraint-feedback up to 3 times.
degrades
Pass 1: ≤3 invariance-feedback retries. Pass 2: one retry with harm-reduction nudge. Pass 3: raise → pipeline_error.
