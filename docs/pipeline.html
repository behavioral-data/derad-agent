Extract & gate
Claim decomposition + check-worthiness
Claude Sonnet 4.6 reasoning_effort=low 45 s timeout
goal
Decompose the tweet into atomic propositions, label each as verifiable / opinion / mixed, mark the one central claim, and decide if any is worth fact-checking.
prompt
"You are the claim-extraction stage. Be CONSERVATIVE on 'verifiable' — spurious fact-checks of opinion are worse than missed ones. Output ExtractionOutput JSON. EXACTLY one claim is_central=true."
tools
Claude reasoning (no external tools). Sees tweet_text + tweet_context (author bio, posted_at, expanded URLs) + per-image OCR/description.
output
ExtractionOutput(claims, overall_state ∈ {checked, no_checkable_claim})
degrades
Parse / timeout → fallback treats whole tweet as one verifiable central claim

4
Verify (Papelo)
Iterative search loop · seed + controller
gpt-5.4-mini + web_search Claude controller ≤5 questions 60 s wall-clock
goal
Gather text evidence about the central claim through a question-at-a-time loop. Skip the controller call for the first iteration — issue the claim text itself as the seed query.
seed prompt
— (no LLM. Query is literally "fact check: {central_claim}".)
controller
"You decide if enough evidence has been gathered. If not, issue the SINGLE next question that closes the biggest gap. Bias toward primary-source / fact-checker / publisher angles. Don't repeat history."
search
"You are a web-search assistant for fact-checking. Always invoke web_search at least once; cite each bullet with the source." (gpt-5.4-mini + native web_search tool. URLs come from url_citation annotations — fabrication impossible — or markdown fallback.)
validate
8-worker thread pool. Annotation URLs short-circuit (server-vouched). Markdown URLs: GET + first 32 KB + <title> + Jaccard ≥ 0.20.
stop when
≥2 fact-checker / reputable-news hits agree · OR no relevant coverage · OR cap hit
output
list of Evidence(question, source_url, snippet, stance="neutral") — stances re-stamped by Stage 4.5
↓
4.5.2
Source tier
Classify every URL into a reliability tier
structural L1 dict · L2 Tables · L3 Claude
goal
Tag every URL as fact-checker / reputable-news / primary-source / aggregator / low-quality / satirical / unknown so the structural verdict rule can read off it.
L1
In-memory: 243 curated domains across the tiers, plus subdomain → parent fallback. ~0 ms.
L2
Azure Tables SourceTierCache — fleet-wide; survives worker / pod restarts. One classification ever per domain.
L3 prompt
"Classify each input domain into ONE tier from this exact set. Be conservative — return 'unknown' if unsure." (Claude · low reasoning · 30 s timeout; only fires on L1+L2 miss; result writes back to both caches.)
output
SourceQualityEntry(url, tier, tier_source, rationale) for every URL
↓
4.5
Reconcile
Three-lens cascade in one Claude call
Claude Sonnet 4.6 reasoning_effort=medium · 4096 thinking 90 s timeout skipped on structural NEI
goal
Reason across the evidence + image OCR + image provenance + author context. Classify every atomic proposition as verified / refuted / disputed / unaddressed. Emit a renderer-facing payload.
prompt
"Use tweet_context actively — parody handles, recycled-event tweets, third-party aggregators. Image is incidental vs image IS the claim. Lens 1 = text-text, Lens 2 = image-text, Lens 3 = cross-modal. Don't invent sources; cite only from source_quality_table. Conservative when evidence is thin."
tools
Pure Claude reasoning. Input prompt carries evidence (snippets capped at 240 chars), source-quality table, image OCR + description + provenance, tweet_context.
skip when
No text evidence AND no image-provenance URL in a credible tier → structurally NEI before reconcile fires; saves ~5–10 s + the Claude call.
output
consolidated_findings · presentation_payload · tone_neutral_justification · evidence_stances (re-stamped per evidence). Stance-count drift emits reconcile_stance_drift metric.
↓
5
Audit
Mechanical verdict check + URL containment
pure Python < 50 ms
goal
Catch reconcile drift before the reply is rendered. Force NotEnoughEvidence if anything is off.
checks
(1) Reconciled verdict matches the structural rule. (2) Every URL in tone_neutral_justification AND load_bearing_evidence_snippet AND primary_sources_to_cite is in source_quality_table. (3) Central proposition lives in exactly one verified/refuted/disputed/unaddressed bucket. (4) Refuted ⇒ counter_fact present.
on fail
Verdict overwritten to NotEnoughEvidence; NEI payload swapped in with reason "evidence retrieved but silent".
↓
6
Freeze
Serialize the FrozenVerdict
Pydantic JSON deep-immutable
goal
Persist a complete, immutable record of everything the backend produced before the renderer sees it. Stages 7+ are forbidden from issuing new searches or model calls that touch evidence.
contents
All claims · all Evidence records · source-quality table · consolidated findings · presentation payload · image OCR + descriptions + provenance · thread context · pipeline git SHA · model + VLM version strings
where
data/freezes/<invocation_id>.json (Blob upload is a follow-up per design §4.7)
↓ stage-7 read only · presentation_payload + tone_neutral_justification
7
Render
Tone-controlled reply · agreeable / neutral / agonistic
Claude Sonnet 4.6 no thinking · 30 s timeout ≤256 X-weighted chars
goal
Write ONE reply tweet in the requested register. Three tones share the same factual payload; only affect varies.
prompts
Three independent system prompts share a common _INPUT_CONTRACT + _HARD_CONSTRAINTS. Tone-specific register guidance: neutral = Bode & Vraga 2020 "straightforward, evidence-first, detached"; agreeable = Lewandowsky Debunking Handbook "affirm impulse, lead with alternative fact, never condescending"; agonistic = Boukes & Hameleers 2022 "pointed, sarcastic — target the claim or source's credibility, never the person."
state-aware
Prompt receives state ∈ {verified, no_sources, no_checkable_claim}. URL inclusion is conditional: verified ⇒ ≥1 URL from primary_sources_to_cite; otherwise no URL.
tools
Reads only RendererView (presentation_payload + tone_neutral_justification + state). Reasoning fields are unreachable by contract.
enforce
Runtime invariance check: rejects renders that introduce URLs outside primary_sources_to_cite, miss the required URL when state=verified, exceed 256 chars, or look like a refusal.
degrades
Pass 1: ≤2 invariance-feedback retries. Pass 2: one retry with harm-reduction nudge. Pass 3: raise → pipeline_error.
↓
