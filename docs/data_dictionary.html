derad-agent · Data Dictionary · 2026-05-18

Data Dictionary

Schema reference for all Azure Table Storage tables written by derad-agent. Produced for IRB review and collaborator onboarding. All tables are in the same Storage Account; the endpoint is DERAD_TABLES_ENDPOINT.

Storage

Azure Table Storage (Standard LRS). Key access disabled; all SDK auth uses DefaultAzureCredential (UAMI in prod, az-cli locally).

Retention

No automatic TTL or deletion policy is enforced today. Rows accumulate indefinitely. The dedup table (Mentions) stores ExpiresAtUtc for a future cleanup pass; other tables are append-only.

PII note

X user IDs and handles are considered pseudonymous identifiers. Fields marked PII should be handled per study protocol. No IP addresses, device IDs, or passwords are stored.

01 · Research log

MentionEvents

One row per mention that passed all guards and entered the pipeline. Written at the terminal point of process_mention() regardless of outcome — the row is always written, even for errors. PartitionKey = YYYY-MM (month of received_at_utc); RowKey = received_at_utc ISO + "_" + mention_id.

Column Type PII Description
mention_id string public X post ID of the @-mention tweet sent by the participant.
parent_id string public X post ID of the tweet the participant wants fact-checked (the reply target).
author_id string PII X numeric user ID of the participant who sent the mention.
author_username string | null PII X handle (screen name) of the participant. Null when ingested via polling (v2 API does not return screen_name without an expansion call).
tone string no Bot personality the participant used: agreeable | neutral | satirical.
parent_text string | null public Full text of the tweet being fact-checked. Truncated to 32 000 chars if longer. Null if parent could not be fetched.
parent_author_id string | null PII X user ID of the author of the tweet being fact-checked.
parent_author_username string | null PII X handle of the author of the tweet being fact-checked.
received_at_utc datetime (UTC) no Timestamp when the webhook or polling worker received the mention.
pipeline_start_utc datetime (UTC) no Timestamp when the background pipeline thread started executing.
reply_posted_utc datetime (UTC) | null no Timestamp when the reply tweet was successfully posted. Null on non-reply outcomes.
pipeline_ms int | null no End-to-end pipeline latency in milliseconds (monotonic clock, webhook receipt → terminal point).
queries json list[string] no Search queries the LLM generated to retrieve Community Notes. Stored as JSON string in Tables; decoded on export.
cited_note_ids json list[string] no Community Note IDs cited in the reply text. Empty list if no notes were found or the reply was empty.
cited_tweet_ids json list[string] no X tweet IDs cited as sources in the reply.
reply_text string | null no Text of the bot's reply tweet. Truncated to 32 000 chars. Null if no reply was generated.
reply_id string | null no X post ID of the reply tweet. Null on non-reply outcomes.
outcome string no Terminal pipeline state. One of: replied, empty_reply, parent_fetch_failed, x_post_error, pipeline_error.
error_class string | null no Python exception class name. Null unless outcome == "pipeline_error".
error_detail string | null no Exception message, truncated to 1 000 chars. Null unless outcome == "pipeline_error".

02 · Filtered traffic log

MentionDrops

One row per mention that was rejected before entering the pipeline. Captures the guard that fired and any structured context. PartitionKey = YYYY-MM; RowKey = received_at_utc ISO + "_" + mention_id (or a timestamp fingerprint when mention_id is absent).

Column Type PII Description
mention_id string | null public X post ID. Null for invalid_payload and unknown_target_user drops where the payload could not be parsed.
author_id string | null PII X numeric user ID of the sender. Null when the payload was malformed.
tone string | null no Bot tone that received the mention. Null for unknown_target_user (routing failed before tone was resolved).
drop_reason string no Guard that fired. One of:
duplicate — already claimed in dedup store;
rate_limit — author exceeded DERAD_RATE_LIMIT_PER_SEC;
daily_cap — DERAD_MAX_MENTIONS_PER_DAY reached for this tone;
self_reply — mention is from the bot itself;
unregistered — author not in DERAD_ALLOWED_AUTHOR_IDS;
no_parent — tweet is not a reply (no in_reply_to);
invalid_payload — malformed webhook body;
unknown_target_user — for_user_id not in TONE_BY_USER_ID.
received_at_utc datetime (UTC) no Timestamp when the mention arrived.
extra json dict no Structured context for the specific drop reason. For rate_limit: {"hits": N}. For invalid_payload: {"why": "...", "type": "..."}. Empty dict otherwise.

03 · Dedup lock store

Mentions

Deduplication lock table. One row per accepted mention ID; the row prevents the same mention from being processed twice (e.g., X webhook redelivery, or poller + webhook race). Written by TablesStore.claim(). PartitionKey is fixed ("mentions"); RowKey = mention_id.

Column Type PII Description
PartitionKey string no Always "mentions". Fixed partition keeps all dedup lookups as single PK+RK point reads.
RowKey string public X post ID of the mention. Unique per claim.
ClaimedAtUtc datetime (UTC) no When the lock was acquired.
ExpiresAtUtc datetime (UTC) no When the lock expires (ClaimedAtUtc + 24 h). Used by a future cleanup job; not enforced by the SDK today.

04 · Rate-limit hit log

RateLimits

Per-author hit log for the sliding-window rate limiter. One row per mention that passed the dedup check; rows are queried with an OData AtUtc ge datetime'...' filter to count hits within the window. PartitionKey = "author:{author_id}"; RowKey = "{timestamp}-{uuid8}".

Column Type PII Description
PartitionKey string PII author:{author_id}. Embeds the X user ID; each author has their own partition for fast per-author windowed queries.
RowKey string no {unix_timestamp_20dec}-{uuid_8hex}. Monotonic-ish for range scans; uuid suffix prevents collisions on simultaneous inserts.
AtUtc datetime (UTC) no Timestamp of this hit. Written as Edm.DateTime so OData chronological comparisons are safe.

05 · Polling cursor store

Cursors

Stores the newest tweet ID seen per bot tone. Only written when DERAD_INGEST_MODE=polling. Inert in webhook mode. PartitionKey is fixed ("cursors"); RowKey = "poll_cursor:{tone}" (one row per bot).

Column Type PII Description
PartitionKey string no Always "cursors".
RowKey string no poll_cursor:{tone}, e.g. poll_cursor:neutral.
Value string no X post ID of the most recently seen mention for this tone. Passed as since_id on the next poll cycle so only newer mentions are fetched.

06 · Export

Exporting data

Use the derad-export CLI to pull tables from Azure Table Storage. Requires az login (or a UAMI environment) and DERAD_TABLES_ENDPOINT.

JSONL (default)
# All accepted mentions from May 2026 onward
derad-export events \
  --since 2026-05-01 \
  --output events.jsonl

# Drops for a date range
derad-export drops \
  --since 2026-01-01 \
  --until 2026-05-31 \
  --output drops.jsonl

# Stream to jq for ad-hoc filtering
derad-export events \
  --since 2026-05-01 | \
  jq 'select(.outcome == "pipeline_error")'
Parquet (pandas + pyarrow)
# Columnar export for analysis
derad-export events \
  --since 2026-01-01 \
  --format parquet \
  --output events.parquet

derad-export drops \
  --since 2026-01-01 \
  --format parquet \
  --output drops.parquet

# Load in Python
import pandas as pd
df = pd.read_parquet("events.parquet")
df.groupby("tone")["outcome"].value_counts()
JSON list fields. The queries, cited_note_ids, cited_tweet_ids (MentionEvents) and extra (MentionDrops) fields are stored as JSON strings in Table Storage and decoded automatically on export. In the raw Azure portal view they appear as queries_json etc. — the _json suffix is stripped by the exporter.
